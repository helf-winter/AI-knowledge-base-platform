# 架构说明

## 1. 总体架构

本项目采用前后端分离架构：

- 前端：Next.js + React，负责知识库管理、专家 Agent 管理、问答交互、任务与记录展示
- 后端：FastAPI + SQLAlchemy，负责知识管理、问答处理、Agent 推荐、自动学习、审计记录、任务流转
- 数据层：关系型数据库存储文档、知识条目、Agent、技能、问答记录、审计日志、任务等信息
- AI 层：通过 DeepSeek 接口完成生成式问答（可选配置）

## 2. 核心业务链路

### 2.1 知识生产

知识来源包括：
- 文档导入
- 手动维护知识元数据
- 问答过程中自动提炼知识缺口

流程：
1. 上传文档
2. 解析文档与分块
3. 生成知识元数据
4. 在问答/自动学习过程中发现知识缺口
5. 生成知识草稿并进入待审核状态

### 2.2 知识管理

平台提供基础知识管理能力：
- 知识条目创建、更新、删除
- 状态区分（可用/不可用/审核中/归档）
- 知识检索
- 审核与归档
- 热度统计与审计查询

### 2.3 知识消费

平台支持两类知识消费方式：

#### 通用问答
- 用于处理常规问题
- 回答简单、直接
- 如果问题更适合某个业务场景，会推荐对应 Agent

#### 专家 Agent 问答
- 通过知识域、知识范围与 skills 形成专业化助手
- 可针对不同业务场景提供更垂直的回答
- Agent 可作为知识库的外化消费入口

### 2.4 Agent 与 Skill 的关系

- `Skill`：平台具备的能力单元，例如知识检索、知识抽取、知识对比、文档总结
- `Agent`：把知识域、技能组合、模型与 prompt 绑定起来形成业务助手

关系可以理解为：
- Skill 是“能力”
- Agent 是“能力组合后的业务角色”

通用问答则负责：
- 常规问题兜底
- 推荐合适的 Agent
- 引导用户进入更专业的业务助手

### 2.5 自动学习闭环

系统会根据问答过程和知识缺口自动形成学习任务：
1. 记录低置信度问答
2. 汇总知识缺口
3. 聚类相似问题
4. 生成知识草稿
5. 进入审核/待办流程
6. 反哺知识库

## 3. 主要模块

### 前端模块
- 知识库管理页面
- 专家 Agent / Skill 页面
- 通用问答浮窗
- 问答记录与操作记录页面
- 待办与任务页面

### 后端模块
- `KnowledgeService`：文档、知识检索、问答核心逻辑
- `KnowledgeAdminService`：知识条目、技能目录、专家 Agent 管理
- `KnowledgeFlywheelService`：自动提炼与学习闭环
- `ConversationService`：问答记录管理
- `AuditService`：审计日志记录与查询
- `AgentRecommender`：通用问答下的 Agent 推荐

## 4. 数据流

### 问答请求
前端发起问答请求 -> 后端检索知识 -> 生成回答 -> 记录问答 -> 记录审计 -> 返回前端

### Agent 推荐
用户使用通用问答 -> 后端识别问题意图 -> 匹配适合的 Agent -> 返回推荐原因 -> 前端展示一键切换入口

### 自动提炼
问答中发现低置信度或高频问题 -> 自动进入知识缺口 -> 聚类 -> 草稿生成 -> 待审核

## 5. 设计原则

- 通用入口负责“兜底 + 分流”
- Agent 负责“垂直业务场景处理”
- Skill 负责“能力沉淀与复用”
- 自动学习负责“形成知识生产闭环”
- 审计与任务页面负责“可追踪、可运营、可排错”
