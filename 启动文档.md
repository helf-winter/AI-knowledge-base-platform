# 本地启动说明

本文档用于说明如何在本地启动 AI 知识库管理平台的完整 Demo。

## 1. 前置条件

请确保本机已安装：

- Python 3.10+
- Node.js 18+
- npm
- PostgreSQL（或项目配置的兼容数据库）

## 2. 一键启动（推荐）

在项目根目录执行：

```powershell
.\start.ps1
```

或者双击/执行：

```bat
start.bat
```

脚本会自动完成：

- 检查 Python 与 npm
- 安装后端依赖
- 首次启动时安装前端依赖
- 自动生成 `frontend/.env.local`
- 同时启动后端 `http://127.0.0.1:8000`
- 同时启动前端 `http://127.0.0.1:3000`
- 启动完成后自动打开浏览器

如果依赖已经安装过，想跳过安装检查：

```powershell
.\start.ps1 -SkipInstall
```

如果不想自动打开浏览器：

```powershell
.\start.ps1 -NoBrowser
```

也可以指定端口：

```powershell
.\start.ps1 -BackendPort 8000 -FrontendPort 3000
```

## 3. 手动启动（备用）

### 3.1 后端依赖

在项目根目录执行：

```bash
pip install -r requirements.txt
```

### 3.2 后端环境变量

在项目根目录创建 `.env` 文件，并配置至少以下内容：

```env
DATABASE_URL=postgresql+psycopg://用户名:密码@localhost:5432/数据库名
DEEPSEEK_API_KEY=你的DeepSeekKey
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_THINKING_ENABLED=true
```

如果暂时不接入 DeepSeek，可以先不填 `DEEPSEEK_API_KEY`，系统会自动回退到检索式回答。

### 3.3 执行数据库迁移

```bash
alembic upgrade head
```

### 3.4 可选初始化脚本

如果项目中有初始化脚本，可以再执行：

```bash
python scripts/init_db.py
```

### 3.5 启动后端服务

```bash
uvicorn app.main:app --reload
```

后端默认地址：

```text
http://127.0.0.1:8000
```

## 4. 前端启动

### 4.1 配置前端环境变量

进入 `frontend` 目录，创建 `.env.local`：

```env
NEXT_PUBLIC_API_BASE=http://127.0.0.1:8000
```

### 4.2 安装依赖

```bash
cd frontend
npm install
```

### 4.3 启动前端

```bash
npm run dev
```

前端默认地址：

```text
http://localhost:3000
```

## 5. Docker Compose 部署

如果需要打包部署到服务器，推荐使用 Docker Compose：

```bash
cp .env.production.example .env.production
```

Windows PowerShell：

```powershell
Copy-Item .env.production.example .env.production
```

修改 `.env.production`：

```env
POSTGRES_PASSWORD=你的数据库密码
DEEPSEEK_API_KEY=你的DeepSeekKey
NEXT_PUBLIC_API_BASE=http://localhost:8000
```

如果部署到云服务器并从其他电脑访问，请把 `NEXT_PUBLIC_API_BASE` 改成服务器可访问地址，例如 `http://服务器IP:8000` 或你的 HTTPS 域名。

启动：

```bash
docker compose --env-file .env.production up -d --build
```

访问：

```text
前端：http://localhost:3000
后端：http://localhost:8000
接口文档：http://localhost:8000/docs
```

查看服务状态：

```bash
docker compose --env-file .env.production ps
```

查看日志：

```bash
docker compose --env-file .env.production logs -f
```

停止：

```bash
docker compose --env-file .env.production down
```

如需连同 PostgreSQL 数据卷一起删除：

```bash
docker compose --env-file .env.production down -v
```

注意：Docker 内部数据库地址使用服务名 `postgres`，不要写成本机的 `127.0.0.1`。当前 `docker-compose.yml` 会自动生成后端容器的 `DATABASE_URL`。

## 6. 演示步骤

1. 打开前端页面并登录
2. 进入首页仪表盘，查看文档、知识条目、任务、最近问答和自动学习概览
3. 进入知识库页面上传或检索文档
4. 点击文档进入详情页，查看 txt、Markdown、CSV、PDF 等不同类型的全文预览
5. 使用右下角 AI 问答助手提问，观察系统自动优先检索知识库、知识不足时切换通用回答
6. 点击回答来源，跳转到对应知识文档并查看关键词高亮
7. 对知识不足的问题选择“扩充”，验证知识追加到相近知识库或创建新知识文档
8. 进入问答记录页面查看多轮问答、Trace ID、引用来源
9. 进入自动学习页面触发知识缺口分析，查看任务状态与重试能力
10. 进入审计日志页面查看上传、问答、扩充、任务等操作记录

## 7. 常见问题

### 7.1 前端提示 401
通常是因为未登录或 token 失效，请重新登录。

### 7.2 接口请求失败
请确认：

- 后端是否已启动
- `NEXT_PUBLIC_API_BASE` 是否正确
- 数据库迁移是否已完成

### 7.3 数据库字段不存在
请先执行：

```bash
alembic upgrade head
```

然后重启后端服务。

### 7.4 前端页面刷新后 500 或白屏
Next.js 开发服务偶尔会因为热更新缓存导致 Client Manifest 错误。可以清理前端缓存后重启：

```bash
cd frontend
rm -rf .next
npm run dev
```

Windows PowerShell 可使用：

```powershell
Remove-Item .next -Recurse -Force
npm run dev
```
