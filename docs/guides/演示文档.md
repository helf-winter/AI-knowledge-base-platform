# AI 企业知识库管理平台全功能演示与验收指南

本文档用于完整检查系统现有功能，也可以作为答辩前的彩排脚本。请严格按照顺序执行，因为后续的发布审核、自动学习和知识整合依赖前面准备的数据。

## 1. 演示范围与时间

### 1.1 完整验收

完整执行预计需要45至70分钟，覆盖：

- Docker与本地运行状态。
- 企业账号登录、首次改密、角色与部门属性。
- 首页仪表盘与角色化导航。
- 文档上传、手动知识、大文件后台解析、重试与继续解析。
- TXT、Markdown、CSV、PDF预览和Chunk查看。
- 关键词、BGE-m3语义、标签和组合筛选。
- 个人、公有、部门知识空间与访问权限。
- 访问申请、发布申请、AI辅助审核和人工审批。
- AI问答、多轮上下文、通用回答、来源引用和知识扩充。
- 问答反馈、自动学习、知识草稿与审核发布。
- 公有知识修改建议、知识整合Agent、专家Agent和Skill。
- 内容管理、问答记录、操作审计和系统设置展示。

### 1.2 答辩精简演示

答辩现场建议从本文档中选择标有“答辩重点”的步骤，控制在10至15分钟。大文件解析、拒绝分支和跨部门验证应提前准备结果，不要现场等待。

## 2. 演示前检查

### 2.1 启动Docker服务

项目位于中文路径时，Docker BuildKit可能报告非ASCII Header错误。已经构建过镜像时直接启动：

```cmd
cd /d D:\code\字节训练营
docker compose -p ai-kb --env-file .env.production up -d
```

确实需要重新构建时，在CMD中执行：

```cmd
set DOCKER_BUILDKIT=0
set COMPOSE_DOCKER_CLI_BUILD=0
docker compose -p ai-kb --env-file .env.production up -d --build
```

检查五个服务：

```cmd
docker compose -p ai-kb --env-file .env.production ps
```

预期：

| 服务 | 预期状态 | 作用 |
|---|---|---|
| `ai-kb-postgres` | `healthy` | PostgreSQL与pgvector |
| `ai-kb-redis` | `healthy` | Celery任务队列 |
| `ai-kb-backend` | `Up` | FastAPI业务服务 |
| `ai-kb-worker` | `Up` | 大文件后台解析 |
| `ai-kb-frontend` | `Up` | Next.js页面 |

访问：

```text
前端：http://127.0.0.1:3000
后端健康检查：http://127.0.0.1:8000/health
API文档：http://127.0.0.1:8000/docs
```

健康检查应返回：

```json
{"status":"ok"}
```

建议准备以下文件，文件名要容易识别：

## 3. 第一阶段：登录、角色与首页

### 3.1 首次登录与修改密码

操作：

1. 打开`http://127.0.0.1:3000/login`。
2. 使用尚未改密的E1004和初始密码`654321`登录。
3. 按住密码框右侧眼睛图标。
4. 松开图标。
5. 登录后进入首次修改密码页。
6. 故意输入错误旧密码，提交一次。
7. 输入两个不一致的新密码，检查前端提示。
8. 输入正确旧密码和一致的新密码，完成修改。
9. 退出后使用新密码重新登录。

预期：

- 按住眼睛时显示密码，松开后恢复隐藏。
- 错误旧密码得到明确中文提示。
- 两次新密码不一致时不能提交。
- 修改成功后可以进入首页，旧密码不能再登录。
- 直接访问`/`时，未登录用户会被重定向到登录页。

### 3.2 普通员工导航与首页自适应

使用E1001登录。

操作：

1. 查看左侧导航。
2. 确认没有“内容管理”“审核管理”“自动学习”等管理员入口。
3. 进入首页仪表盘。
4. 检查知识文档、任务状态和最近问答区域。
5. 将浏览器窗口缩窄，观察卡片重排。
6. 手动输入`http://127.0.0.1:3000/admin`。

预期：

- 普通员工界面没有管理员菜单标题和空白占位。
- 首页使用同一套页面代码，根据角色隐藏管理员模块并自适应铺满。
- 普通员工不能进入管理员页面。
- 即使绕过前端导航直接访问管理员接口，后端也返回无权限。

### 3.3 管理员导航

退出E1001，使用admin登录。

预期：

- 可以看到内容管理、自动学习、操作审计等管理入口。
- 首页比普通员工多出治理相关入口。
- 管理员身份来自用户角色属性，不需要创建第二个同名管理员账户。

## 4. 第二阶段：知识生产与文档解析

以下操作使用E1001。

### 4.1 上传Markdown文档（答辩重点）

1. 进入“知识库”`/documents`。
2. 在“上传文档”中选择`VPN使用指南.md`。
3. 点击“开始上传”。
4. 观察是否跳转到文档详情。
5. 等待状态从排队或解析中变为解析完成。

预期：

- 上传成功后立即创建文档记录，不长时间阻塞页面。
- 文档默认属于E1001的个人知识空间。
- 解析后产生全文、Chunk和BGE-m3向量。
- 管理员和其他普通员工不能直接查看这份未发布个人知识。

### 4.2 手动创建知识（答辩重点）

返回知识库，在“手动记录知识”填写：

```text
标题：VPN申请经验
知识类别：IT与技术
适用人员：全公司
业务用途：减少VPN申请咨询成本
标签：VPN,权限申请,远程办公
正文：
## 适用场景
员工需要在公司外访问内网系统。

## 申请步骤
1. 进入IT服务台。
2. 填写申请原因和使用期限。
3. 等待直属负责人和IT管理员审批。
```

点击保存。

预期：

- 生成Markdown个人知识文档。
- 结构化Tag写入标签表，而不是只存在正文中。
- 文档可以像上传文档一样提交发布审核。

### 4.3 多格式预览

依次验证：

| 格式 | 预期表现 |
|---|---|
| TXT | 保留换行和基本原始格式 |
| Markdown | 标题、列表、粗体等得到渲染 |
| CSV | 以表格方式展示，而不是一整行文本 |
| PDF | 优先内嵌原始PDF，原文件不可用时显示提取文本 |

进入每个文档详情页，检查：

- “返回知识库”按钮可用。
- 文档元数据可见。
- 全文预览与文件类型匹配。
- Chunk编号、页码和正文可见。
- 从问答来源跳转时，查询关键词可以高亮。

### 4.4 文档列表交互

在“知识文档”列表中：

1. 点击文档行空白处。
2. 检查右侧“文档摘要”是否更新。
3. 点击文档名称或右侧箭头。

预期：

- 空白处只切换摘要，不进入详情。
- 文档名和箭头进入详情页。

### 4.5 大文件后台解析（完整验收）

1. 上传一个超过`BACKGROUND_PARSE_THRESHOLD_MB`的大PDF。
2. 观察状态：排队中、解析中、向量化、完成或失败。
3. 上传后关闭AI浮窗、切换页面或退出当前账号。
4. 使用管理员或原账号重新进入文档详情查看状态。

预期：

- 上传请求快速返回，文件进入Redis队列。
- Celery Worker在后台继续处理，退出登录不会中断。
- 进度、阶段和错误信息保存在PostgreSQL。
- 页面可以查看任务状态。

后台检查：

```cmd
docker compose -p ai-kb --env-file .env.production logs --tail 100 worker
```

### 4.6 失败、重试与继续解析

如果已有失败任务：

1. 在文档详情或任务页面查看失败原因。
2. 点击“重试”或“继续解析”。
3. 观察重试计数、阶段和进度变化。

预期：

- 已生成的Chunk不会重复写入。
- 缺失向量会被补齐。
- 失败后不需要重新上传原文件。

如果没有自然失败任务，正式答辩只展示已有任务状态，不要人为破坏服务。

### 4.7 删除个人文档

上传一份专门用于删除测试的小文件：`待删除测试.txt`。

1. 在文档摘要或详情中点击删除。
2. 确认删除提示。
3. 刷新列表并检索文件名。

预期：文档不再出现在当前用户可用知识中。不要使用正式演示知识做删除测试。

## 5. 第三阶段：知识检索

### 5.1 关键词检索

在“知识检索”输入：

```text
VPN
```

预期：

- 返回标题或正文包含VPN的知识。
- 结果显示来源文档、摘要和相关分数。
- 只显示当前用户有权访问的内容。

### 5.2 真实语义检索（答辩重点）

输入不直接包含原文字面的语义问题，例如：

```text
员工在家如何访问公司内网？
```

预期：

- 即使文档主要使用“VPN”“远程办公”等词，也能召回相关知识。
- 说明查询由本地BGE-m3生成1024维向量，通过pgvector计算相似度。

### 5.3 标签与组合筛选

依次选择：

- 知识类别：IT与技术。
- 标签：VPN或远程办公。
- 知识空间：个人知识。
- 文件类型：Markdown。
- 人员范围：全公司。

预期：

- 筛选条件会随检索请求发送到后端。
- 结果同时满足选定条件。
- 点击清空筛选后恢复。
- 下拉项只来自当前用户可见知识，不泄露其他人的个人标签。

### 5.4 跨用户个人知识隔离

1. 记录E1001创建的`VPN申请经验`文档名。
2. 退出E1001，登录E1002。
3. 搜索完整标题、标签和正文关键词。
4. 尝试使用已知文档详情URL直接访问。

预期：

- E1002搜索不到E1001未发布的个人知识。
- E1002不能直接打开该详情。
- 管理员也不能因为admin角色绕过个人知识隐私。

## 6. 第四阶段：AI问答助手

### 6.1 浮窗交互

1. 拖动右下角AI助手悬浮按钮。
2. 确认拖动不会误触发打开。
3. 打开浮窗后拖动边框调整大小。
4. 输入问题后立即点击关闭。
5. 等待回答完成提示，再重新打开。

预期：

- 拖动与点击被正确区分。
- 面板可以调整大小。
- 关闭浮窗不终止后端回答。
- 回答完成后出现提示，重新打开仍能看到结果。

### 6.2 知识库问答与Markdown渲染（答辩重点）

提问：

```text
VPN使用步骤是什么？请分步骤回答。
```

预期：

- 回答正文完整，不只显示标题。
- Markdown标题、加粗和列表得到渲染。
- 回答优先使用知识库内容。
- 返回引用序号、来源文档和Trace ID。
- 只展示实际命中的来源，不展示Agent绑定的所有文档。

### 6.3 来源跳转与高亮

1. 点击回答下方带序号的来源文档。
2. 检查是否进入正确文档详情。
3. 检查查询关键词是否高亮。
4. 点击“返回知识库”。

预期：来源序号与回答正文中的引用一致，文档跳转准确。

### 6.4 多轮上下文

连续提问：

```text
第一问：VPN使用步骤是什么？
第二问：第二步需要准备什么？
第三问：如果审批不通过怎么办？
```

预期：后两问能够结合近期对话理解“第二步”和“审批”的指代。

### 6.5 知识不足与通用回答兜底（答辩重点）

提问一条知识库明显没有覆盖的问题，例如：

```text
请解释一项当前知识库没有记录的新技术概念。
```

预期：

- 系统不会只回复“不知道”。
- 知识不足时切换到通用DeepSeek回答。
- 回答末尾询问是否扩充该知识。
- 用户可以选择“暂不”或“扩充”。

### 6.6 AI问答扩充个人知识

1. 对上一条通用回答点击“扩充”。
2. 返回知识库，按标题或关键词搜索。
3. 检查新增知识的所有者和空间。

预期：

- 内容会优先整理到相近的当前用户个人知识，必要时新建文档。
- 不会每次无条件创建新知识库文档。
- 新知识默认属于当前用户个人空间，不直接影响全公司。

### 6.7 关闭、刷新、切页与账号隔离

验证：

1. 关闭并重新打开浮窗，消息仍在。
2. 切换系统页面，消息仍在。
3. 刷新页面，当前用户缓存消息仍在。
4. 退出E1001并登录E1002。

预期：E1002看不到E1001的前端消息缓存，消息按用户隔离。

## 7. 第五阶段：问答记录与质量反馈

### 7.1 问答记录

使用当前账号进入`/conversations`。

检查：

- 多条问答历史同时存在。
- 显示问题、完整回答、置信度、时间、来源和Trace ID。
- 可以按session_id或trace_id筛选。
- 页面不再提供不合理的“去操作记录”跳转。

### 7.2 有帮助反馈

对一条回答点击“有帮助”。

预期：提示反馈已记录，不出现`Failed to fetch`。

### 7.3 无帮助反馈

对另一条低质量回答点击“无帮助”。

预期：

- 反馈只允许提交到当前用户自己的问答。
- 系统提示反馈会纳入自动学习。
- 后续管理员可以在知识缺口中看到相关问题。

记录该条问答的Trace ID，后面用于审计筛选。

## 8. 第六阶段：个人知识发布与公有复用

### 8.1 提交公有发布申请（答辩重点）

使用E1001：

1. 找到自己的`VPN申请经验`。
2. 点击“发布审核”或“提交公有审核”。
3. 从下拉框选择知识类别，例如“IT与技术”。
4. 选择可访问人员工作类别，例如“全公司”。
5. 填写发布理由和业务用途。
6. 提交申请。
7. 在“我的发布申请”确认状态为待审核。

预期：

- 类别和人员范围使用下拉选项，不要求随意输入。
- 申请提交后原文档仍是E1001个人知识。
- E1002在审核前搜索不到该知识。
- 已有待审核申请不能重复提交。

### 8.2 管理员发布审核（答辩重点）

退出E1001，登录admin，进入`/admin`。

1. 按`pending`筛选发布申请。
2. 查看申请人、文档预览、类别、人员范围、发布理由和业务用途。
3. 输入审核意见。
4. 点击通过。

预期：

- 管理员只有在用户主动提交后才能看到必要预览。
- 管理员不能从普通知识列表随意发布他人个人知识。
- 审核通过后创建公有知识引用，不复制或删除原始文档。
- 已审核申请不能重复审核。

### 8.3 公有知识复用

1. 登录E1002。
2. 搜索`VPN申请经验`。
3. 打开文档详情。

预期：

- E1002可以通过公有引用搜索和阅读。
- 页面可说明原始文档仍属于个人空间，当前通过公有引用访问。
- AI问答可以召回该公有知识。

### 8.4 发布拒绝分支

使用另一份测试个人知识重复8.1，但管理员选择拒绝。

预期：

- 原知识继续保持个人知识。
- 其他员工仍然搜索不到。
- 用户在“我的发布申请”看到`rejected`和审核意见。

### 8.5 公有知识下架

管理员在公有知识管理区域选择刚发布的测试公有引用并下架。

预期：

- 公有引用失效，其他员工不再通过公有范围搜索到它。
- E1001原始个人文档没有被删除，仍可由所有者访问。

## 9. 第七阶段：文档访问申请与AI辅助审核

该流程需要知识库中预先存在一份“需申请”的受限文档。知识列表或检索结果会显示“需申请”标记。如果当前数据库没有受限样本，请在答辩前准备，不要现场临时改正式数据。

### 9.1 普通员工提交访问申请

1. 使用E1001搜索受限文档。
2. 点击“申请访问”。
3. 填写申请原因、业务用途和预期使用期限。
4. 提交。
5. 在“我的申请”查看状态。

预期：

- 未授权时不能打开正文和原文件。
- 提交后状态为`pending`。
- 重复待审核申请不会创建多条。
- 个人知识不能通过访问申请绕过所有者隔离。

### 9.2 管理员AI辅助审核

1. 使用admin进入“权限申请审核”。
2. 按`pending`筛选。
3. 选择申请，查看申请人、部门、岗位、文档、原因和业务用途。
4. 点击“生成AI建议”。

预期：AI输出：

- `suggestion`：建议通过或拒绝。
- `risk_level`：风险等级。
- `reason`：分析理由。

强调：AI不会自动改变申请状态。

### 9.3 人工通过

管理员输入审核意见并通过。

预期：

- 申请状态变为`approved`。
- 系统创建`document_access_grant`。
- E1001重新登录或刷新后可以打开该文档。
- 已审核申请不能再次审批。

### 9.4 人工拒绝

使用另一条访问申请验证拒绝：

- 状态变为`rejected`。
- 不创建访问授权。
- 用户仍不能访问文档。
- “我的申请”显示审核意见。

### 9.5 DeepSeek不可用降级

完整技术验收可在测试环境暂时移除DeepSeek Key并重启后端，然后触发AI审核建议。

预期：规则fallback仍能返回建议、风险和理由，审核流程不中断。正式答辩不建议现场修改环境变量。

## 10. 第八阶段：公有知识修改建议

## 20. 当前边界与答辩表述

以下能力不要夸大：

- 扫描版PDF、复杂表格和重水印文档仍需增强OCR和版面分析。
- 部门知识读取隔离已实现，但部门发布入口尚未完整产品化。
- 系统设置页尚不是可持久化的生产配置中心。
- 当前使用轻量重排，Cross-Encoder属于后续优化。
- 原始文件主要保存在本地或Docker数据卷，尚未迁移MinIO。
- Worker已独立运行，但生产环境的多Worker、优先级和监控仍可增强。
- Skill目前是系统内部能力，尚未封装为外部Agent插件协议。
- 知识版本已有状态与审计基础，完整diff、回滚和灰度发布仍待完善。

推荐总结话术：

> 本项目完成了企业知识生产、治理、检索和消费的核心闭环。用户上传或创建的内容默认属于个人知识，经过人工审核后才能进入公有范围；系统使用BGE-m3、pgvector、关键词和标签进行权限内混合检索，再通过RAG和DeepSeek生成带来源的回答。用户反馈能够形成知识缺口，AI生成草稿后由管理员审核发布。项目还实现了专家Agent、Skill、知识整合、后台解析、操作审计和Docker部署，使知识能够在安全、可追踪的流程中持续沉淀和复用。

## 21. 快速故障定位

| 现象 | 优先检查 |
|---|---|
| 页面打不开 | Docker Desktop是否`Engine running`，Frontend是否Up |
| `Failed to fetch` | Backend健康检查、API Base、浏览器控制台 |
| 大文件一直排队 | Redis和Worker状态、Worker日志 |
| 文档一直解析中 | 任务阶段、错误信息、原文件是否可读 |
| PDF只有“未提取到文本” | 是否扫描版、图片型或重水印PDF |
| 语义检索无结果 | BGE-m3挂载路径、向量维度、Chunk是否有embedding |
| AI回答失败 | DeepSeek Key、网络、额度与fallback日志 |
| 普通用户看不到管理页面 | 正常的角色权限表现 |
| Docker构建Header含非ASCII | 中文项目路径触发BuildKit问题，使用兼容变量或迁移英文路径 |
| 本地历史数据与Docker不同 | 本机PostgreSQL和Docker数据卷是两套独立数据库 |
