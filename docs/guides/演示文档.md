# AI 企业知识库管理平台演示文档

本文档用于课堂验收和答辩演示。建议控制在8至12分钟，围绕“知识生产、知识管理、知识消费、自动学习闭环”展开。

## 1. 演示目标

通过普通员工和管理员两个角色证明：

- 知识可以通过文档、手动录入和AI使用过程产生。
- 个人、公有和部门知识能够安全隔离。
- 系统支持关键词、BGE-m3语义检索和标签筛选。
- AI问答能够引用来源，知识不足时通用兜底。
- 个人知识经过人工审核后才能进入公有知识库。
- 低质量回答能够形成知识缺口和待审核草稿。
- 相似文档可以由知识整合Agent生成整合建议。

## 2. 演示前准备

### 2.1 启动系统

本地双击`start.bat`，或使用Docker：

```powershell
docker compose -p ai-kb --env-file .env.production up -d
docker compose -p ai-kb --env-file .env.production ps
```

确认：

```text
前端：http://127.0.0.1:3000
后端健康检查：http://127.0.0.1:8000/health
API文档：http://127.0.0.1:8000/docs
```

### 2.2 账号准备

全新数据库会初始化：

```text
管理员：admin / 123456
普通员工：E1001、E1002、E1003 / 654321
```

首次登录需要修改密码；如果数据卷已经使用过，应使用实际修改后的密码。答辩前务必提前验证账号。

### 2.3 演示数据

建议准备：

- 一个小型Markdown或文本PDF，用于现场上传。
- 两份内容相近的VPN文档，用于知识整合演示。
- 一个CSV文件，用于表格预览。
- 已经解析完成的较大PDF，用于展示后台任务状态，避免现场等待。

不要在答辩现场上传80MB以上文件后等待完整向量化。

## 3. 推荐演示路线

### 3.1 企业登录与角色权限

1. 使用普通员工登录。
2. 展示普通员工侧边栏没有内容管理和审核入口。
3. 直接访问`/admin`，证明后端和页面均进行权限控制。
4. 切换管理员账号，展示管理入口。

讲解：

> 系统使用同一套页面代码，根据用户角色动态展示功能；后端接口仍会独立校验权限，避免只隐藏菜单造成越权。

### 3.2 首页知识工作台

展示：

- 当前用户可访问文档数量。
- 解析任务状态。
- 最近问答。
- 管理员可见的自动学习和治理入口。

普通员工和管理员使用同一布局，但数据和管理模块按权限自适应。

### 3.3 知识生产与后台解析

1. 普通员工进入`/documents`上传小型PDF或Markdown。
2. 说明上传内容默认进入个人知识空间。
3. 查看排队、解析、向量化和完成状态。
4. 进入详情页查看原文预览、Chunk和解析任务。
5. 展示CSV表格预览或PDF原文预览。

讲解：

> 小文件可以快速解析，大文件进入Redis和Celery后台队列。用户退出登录不会中断Worker，状态和检查点保存在数据库中，失败后可以继续解析。

### 3.4 混合检索与标签筛选

1. 使用与文档字面表达不同、但语义相近的问题检索。
2. 展示BGE-m3和pgvector能够召回相关文档。
3. 选择知识类别、标签、空间或文件类型进一步筛选。
4. 切换另一个员工，证明其无法看到前一位员工的个人知识和标签。

讲解：

> 语义检索解决“意思相近”，关键词检索补充精确匹配，标签筛选负责业务范围约束，三者都在权限过滤后执行。

### 3.5 AI问答与来源追踪

1. 打开全局AI问答助手。
2. 提问“VPN使用步骤是什么？”。
3. 展示Markdown渲染、多轮上下文和引用序号。
4. 点击来源文档，跳转详情并高亮查询内容。
5. 展示Trace ID和问答记录。

讲解：

> 系统先检索当前用户有权访问的企业知识，再由DeepSeek组织答案；回答保留来源和Trace ID，不是让模型无依据自由生成。

### 3.6 知识不足与个人补充

1. 提问知识库未覆盖的问题。
2. 展示系统自动切换到通用回答，而不是只回复“不知道”。
3. 点击“扩充”。
4. 回到知识库查看新增或追加的个人知识。

强调：AI问答补充默认属于当前用户个人知识，不会直接进入全公司公有知识库。

### 3.7 个人知识发布审核

1. 普通员工选择自己的个人知识，提交到公有知识库。
2. 从下拉框选择目标类别和可访问人员工作类别。
3. 填写发布理由与业务用途。
4. 切换管理员，查看申请和AI辅助分析。
5. 管理员人工通过。
6. 切换另一位符合条件的员工，搜索并访问该知识。

验证点：

- 审核前其他员工搜索不到。
- 管理员不能随意发布用户未提交的个人知识。
- AI只提供建议，最终审批由管理员完成。
- 审核通过后创建公有引用，原个人文档仍然保留。

### 3.8 公有知识建议与访问申请

可根据答辩时间二选一：

- 普通员工对公有知识提交问题或修改建议，管理员接受、拒绝或要求补充。
- 普通员工对无权文档提交访问申请，管理员查看AI建议、风险等级和理由后人工审批。

讲解：

> 普通员工不能直接修改公有知识，但可以参与知识治理；AI是审核助手，不是自动审批者。

### 3.9 自动学习闭环

1. 对一条回答点击“无帮助”。
2. 管理员进入`/tasks`查看知识缺口。
3. 生成AI知识草稿。
4. 管理员编辑、审核并发布。
5. 再次提问类似问题，验证新知识能够被召回。

闭环：

```text
知识消费
-> 质量反馈
-> 缺口发现
-> AI草稿
-> 人工审核
-> 公有知识
-> 后续问答复用
```

### 3.10 专家Agent与Skill

1. 进入`/skills`。
2. 展示知识检索、文档总结、流程提取、知识对比、缺口发现等Skill。
3. 选择知识范围生成专家配置。
4. 展示专家名称、职责、适用问题、回答边界和绑定Skill。
5. 创建专家后通过AI助手使用其知识范围。

### 3.11 知识整合Agent

1. 管理员扫描相似知识。
2. 展示BGE-m3/pgvector发现的相似文档。
3. 查看DeepSeek生成的整合标题、正文、来源编号和冲突说明。
4. 管理员人工审核并创建整合文档。
5. 验证原始文档仍然存在。

### 3.12 问答记录与审计

- `/conversations`：展示问题、答案、置信度、来源、反馈和Trace ID。
- `/audit`：展示上传、问答、审核和治理等关键操作。

说明二者区别：问答记录用于复盘回答质量，审计日志用于追踪系统操作。

## 4. 答辩讲解词

> 本项目实现了一个面向企业内部的AI知识库管理平台。知识可以通过文档上传、手动录入和AI使用过程产生；系统通过个人、公有和部门知识空间控制访问范围，并使用BGE-m3、pgvector、关键词和标签实现混合检索。用户可以通过AI问答、专家Agent和Skill消费知识。低质量回答会形成知识缺口，AI生成草稿后由管理员审核发布。相似文档还可以由知识整合Agent生成带来源的整合建议。因此，它不仅能够回答问题，还能够让知识在安全、可审核的流程中持续沉淀和复用。

## 5. 验收清单

- [ ] 普通员工与管理员菜单和接口权限不同。
- [ ] 用户A个人知识对用户B和管理员默认不可见。
- [ ] 文档可以上传、解析、预览并生成Chunk和向量。
- [ ] Redis和Worker能够消费大文件解析任务。
- [ ] 关键词、语义检索和标签筛选均可展示。
- [ ] AI回答显示来源，来源可以跳转并高亮。
- [ ] 知识不足时自动通用回答，并可沉淀为个人知识。
- [ ] 发布审核通过前，其他用户无法搜索个人知识。
- [ ] AI审核建议不自动执行审批。
- [ ] 无帮助反馈能够形成知识缺口和草稿。
- [ ] 专家Agent能够绑定知识范围和Skill。
- [ ] 知识整合建议带来源，审核后生成新文档且原文保留。
- [ ] 问答记录和审计日志可以区分并正常查看。

## 6. 不建议现场演示

- 不现场等待80MB以上PDF完整解析。
- 不使用扫描版或严重水印PDF验证文本抽取。
- 不在未确认DeepSeek网络和余额时临时生成大量草稿。
- 不执行`docker compose down -v`。
- 不展示或提交`.env`中的真实密钥。
