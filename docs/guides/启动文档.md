# AI 企业知识库管理平台启动文档

本文档说明本地开发和 Docker Compose 两种完整启动方式。当前系统包含前端、后端、PostgreSQL/pgvector、Redis 和 Celery Worker。

## 1. 服务组成

| 服务 | 作用 | 默认端口 |
|---|---|---|
| Next.js | 前端页面 | 3000 |
| FastAPI | 后端接口与业务服务 | 8000 |
| PostgreSQL + pgvector | 业务数据与语义向量 | 本地通常5432，Docker当前映射5433 |
| Redis | Celery任务队列 | 6379 |
| Celery Worker | 大文件后台解析 | 无外部端口 |

本地开发时可以使用本机PostgreSQL；Docker模式会启动独立PostgreSQL容器，两套数据库数据互不自动同步。

演示账号中的“部门”是用户属性，不是独立角色。部门知识通过当前用户所属部门判断是否可见。

## 2. 本地启动

### 2.1 前置条件

- Conda环境：`knowledge-base`
- Node.js 18+
- PostgreSQL，并启用`vector`扩展
- Redis（大文件后台解析需要）
- 本地BGE-m3模型，默认路径`D:/code/models/bge-m3`

确认pgvector：

```sql
SELECT * FROM pg_extension WHERE extname = 'vector';
```

### 2.2 环境变量

项目根目录`.env`至少包含：

```env
DATABASE_URL=postgresql+psycopg://postgres:数据库密码@localhost:5432/knowledge_base
REDIS_URL=redis://localhost:6379/0

DEEPSEEK_API_KEY=你的DeepSeekKey
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash

EMBEDDING_PROVIDER=bge
EMBEDDING_MODEL_PATH=D:/code/models/bge-m3
EMBEDDING_DIMENSION=1024
BACKGROUND_PARSE_THRESHOLD_MB=20
MAX_UPLOAD_SIZE_MB=100
```

不要将真实密码或API Key提交到Git。

### 2.3 首次安装依赖

```powershell
conda activate knowledge-base
python -m pip install -r requirements.txt

cd frontend
npm install
cd ..
```

后续启动不会默认重复下载依赖。仅依赖文件发生变化时执行：

```powershell
.\start.ps1 -InstallDeps
```

### 2.4 数据库迁移

首次启动或拉取到新迁移后执行：

```powershell
conda activate knowledge-base
alembic upgrade head
```

`alembic upgrade head`用于升级已有数据库结构；`python scripts/init_db.py`只用于基础建表，不能替代迁移。

### 2.5 一键启动

直接双击`start.bat`，或者在项目根目录执行：

```powershell
.\start.bat
```

实际流程：

```text
start.bat
  -> 调用start.ps1
  -> 检查Conda与Node.js
  -> 使用knowledge-base环境启动FastAPI
  -> Redis可用时启动Celery Worker
  -> 启动Next.js
  -> 检查前后端健康状态
  -> 自动打开浏览器
```

常用参数：

```powershell
# 不自动打开浏览器
.\start.ps1 -NoBrowser

# 不启动Worker
.\start.ps1 -NoWorker

# 指定端口
.\start.ps1 -BackendPort 8000 -FrontendPort 3000
```

如果Redis未启动，小文件仍可同步解析，但大文件后台任务无法消费。

### 2.6 手动启动

后端终端：

```powershell
conda activate knowledge-base
alembic upgrade head
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

Worker终端（先启动Redis）：

```powershell
conda activate knowledge-base
celery -A app.worker.celery_app worker --pool=solo --loglevel=info
```

前端终端：

```powershell
cd frontend
$env:NEXT_PUBLIC_API_BASE="http://127.0.0.1:8000"
npm run dev -- --hostname 127.0.0.1 --port 3000
```

## 3. Docker Compose启动

### 3.1 准备生产环境变量

```powershell
Copy-Item .env.production.example .env.production
```

至少检查：

```env
POSTGRES_PASSWORD=数据库密码
DEEPSEEK_API_KEY=DeepSeekKey
NEXT_PUBLIC_API_BASE=http://localhost:8000
EMBEDDING_MODEL_HOST_PATH=D:/code/models/bge-m3
MAX_UPLOAD_SIZE_MB=100
```

云服务器部署时，`NEXT_PUBLIC_API_BASE`必须改为浏览器可以访问的后端地址。

### 3.2 构建并启动

```powershell
docker compose -p ai-kb --env-file .env.production up -d --build
```

该命令会构建前端、后端和Worker镜像，并启动：

```text
postgres -> redis -> backend -> worker -> frontend
```

后端容器启动时会自动执行`alembic upgrade head`。

### 3.3 检查状态

```powershell
docker compose -p ai-kb --env-file .env.production ps
```

正常情况下应看到五个服务：

- `ai-kb-postgres`：healthy
- `ai-kb-redis`：healthy
- `ai-kb-backend`：Up
- `ai-kb-worker`：Up
- `ai-kb-frontend`：Up

健康验证：

```text
前端：http://127.0.0.1:3000
后端：http://127.0.0.1:8000
API文档：http://127.0.0.1:8000/docs
健康检查：http://127.0.0.1:8000/health
```

查看日志：

```powershell
docker compose -p ai-kb --env-file .env.production logs -f
```

### 3.4 停止与再次启动

停止并保留数据库和上传文件：

```powershell
docker compose -p ai-kb --env-file .env.production down
```

再次启动已有容器：

```powershell
docker compose -p ai-kb start
```

不要随意执行`down -v`，它会删除PostgreSQL和上传文件数据卷。

### 3.5 什么时候需要重建镜像

| 改动 | 操作 |
|---|---|
| Python或前端代码 | `up -d --build` |
| `requirements.txt`、`package.json`或Dockerfile | `up -d --build` |
| 仅修改数据库数据 | 不需要重建 |
| 新增Alembic迁移 | 重建或重启后端，确认迁移执行成功 |
| 仅重新运行已有容器 | `docker compose -p ai-kb start` |

Docker后端镜像明确安装CPU版PyTorch，避免为CPU部署下载无用CUDA依赖。

## 4. 常见问题

### 4.1 BuildKit提示非ASCII Header

如果项目位于中文路径，Docker Desktop可能报告：

```text
x-docker-expose-session-sharedkey contains non-printable ASCII characters
```

优先将项目迁移到纯英文路径。临时可在当前PowerShell执行：

```powershell
$env:DOCKER_BUILDKIT='0'
$env:COMPOSE_DOCKER_CLI_BUILD='0'
docker compose -p ai-kb --env-file .env.production up -d --build
```

### 4.2 前端显示Failed to fetch

依次确认：

1. `http://127.0.0.1:8000/health`是否返回`ok`。
2. `NEXT_PUBLIC_API_BASE`是否正确。
3. 后端容器是否反复重启。
4. 浏览器访问地址和前端构建时API地址是否一致。

### 4.3 大文件一直排队

检查Redis和Worker：

```powershell
docker compose -p ai-kb --env-file .env.production ps
docker compose -p ai-kb --env-file .env.production logs --tail 100 worker redis
```

### 4.4 扫描版PDF没有正文

扫描版、纯图片或重水印PDF可能无法提取文本。当前系统会保留原文件并提示未提取到有效内容，后续需要接入更完整的OCR流程。

### 4.5 Docker与本地历史数据不同

本地启动通常连接本机PostgreSQL，Docker连接`ai-kb_postgres_data`数据卷。它们是两套独立数据库，因此账号、文档和问答记录不会自动同步。
