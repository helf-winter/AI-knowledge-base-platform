# 动态专家 Agent 执行设计

## 目标

把“专家助手配置档案”升级为真正影响问答执行的动态专家 Agent。用户可以主动选择专家，也可以让系统自动推荐并执行最匹配的专家。

## 执行规则

- 专家 Agent 继续存储在 `expert_agent_profiles` 中，不为每个专家生成独立 Python 类。
- `agent_id` 明确传入时，系统使用该专家执行；未传入时，由推荐器根据问题自动选择最匹配专家。
- 如果推荐器无法匹配，使用通用知识库专家。
- 专家的 `knowledge_scope_json` 支持限制文档 ID、知识类型或关键词范围。
- 专家的 `skills_json` 记录本次允许能力；当前问答主流程至少执行 `knowledge_search`，并把允许能力写入 trace/metadata。
- DeepSeek 提示词包含专家名称、知识域、描述和知识范围说明。
- 回答结束元数据返回实际使用的 Agent、选择原因和启用 Skills，前端展示给用户。

## 边界

- 本轮不生成独立代码文件式 Agent。
- 本轮不实现复杂多 Agent 协作，只实现配置驱动的单专家执行。
- 若专家知识范围无命中，沿用现有通用回答兜底和知识扩充逻辑。

## 验证

- 指定专家时使用该专家并应用知识范围过滤。
- 未指定专家时可自动推荐并返回实际使用的专家。
- DeepSeek 消息包含专家身份信息。
- 前端问答助手展示实际使用专家和推荐原因。
