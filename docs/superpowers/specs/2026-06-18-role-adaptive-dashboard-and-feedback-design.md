# 角色自适应首页与问答反馈修复设计

## 目标

继续使用同一套首页和侧栏代码，根据用户的 `roles` 动态展示和排版。普通员工页面不保留管理员区域造成的空位，同时修复新会话记录无法提交反馈的问题。

## 界面设计

- 普通员工首页仅展示现有内容：知识文档、任务状态、最近问答、最近文档和任务队列。
- 普通员工的两张概览卡各占半宽；最近问答占满整行；最近文档与任务队列保持双列。
- 管理员和审核员继续显示四张概览卡、自动学习区域和原有双列布局。
- “管理入口”分组标题仅对管理员和审核员显示；普通员工仍可看到不属于管理功能的“问答记录”。
- 移动端统一使用单列布局。

## 反馈数据设计

当前反馈表仍通过外键关联旧版 `sessions` 和 `answers`，而问答记录已经迁移到 `conversation_turns`。新反馈写入时因此违反旧外键约束。

- 新迁移移除反馈表指向旧表的两个外键，保留现有字段和历史数据。
- 服务层以 `ConversationTurn.turn_id` 校验 `answer_id`，并确认该问答属于当前用户。
- 找不到问答或尝试评价他人问答时返回中文业务错误。
- 接口捕获业务异常并返回稳定的 HTTP 错误响应；前端展示可读错误，而不是裸露 `failed to fetch`。

## 验证

- 源码回归测试覆盖角色自适应网格和侧栏标题。
- 服务测试覆盖本人反馈成功、问答不存在、评价他人问答三种情况。
- Alembic 升级后检查旧外键已移除。
- 运行 Python 编译、单元测试和前端生产构建。
